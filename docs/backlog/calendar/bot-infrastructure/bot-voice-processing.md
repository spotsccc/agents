# Voice Message Processing

> Приём голосовых сообщений из Telegram, транскрипция через OpenAI Whisper API.

---

## Описание

Бот принимает голосовые сообщения от пользователя, скачивает аудио-файл через Telegram Bot API, отправляет на транскрипцию в OpenAI Whisper API и передаёт полученный текст дальше в pipeline (LLM integration).

---

## Бизнес-сценарии

### Happy Path

1. Привязанный пользователь отправляет голосовое сообщение
2. Бот скачивает `.ogg` файл через Telegram Bot API
3. Бот отправляет файл в Whisper API
4. Whisper возвращает транскрипцию
5. Текст передаётся в LLM для обработки (как обычное текстовое сообщение)

### Слишком длинное сообщение

1. Пользователь отправляет голосовое сообщение > 25MB
2. Бот проверяет размер файла до отправки в Whisper
3. Бот отвечает: TODO (сообщение о превышении лимита)

### Ошибка транскрипции

1. Whisper API возвращает ошибку / таймаутит
2. Бот отвечает: TODO (сообщение об ошибке, предложение повторить или отправить текстом)

### Превышение дневного лимита

1. Пользователь превысил дневной лимит голосовых сообщений
2. Бот отвечает: TODO (сообщение о лимите, когда сбросится)

## Edge Cases

- Voice message с нулевой длительностью
- Whisper возвращает пустую строку (тишина / неразборчиво)
- Telegram Bot API недоступен при скачивании файла
- Пользователь отправляет несколько voice messages подряд (параллельная обработка)
- Аудио-файл повреждён / невалидный формат
- Whisper API rate limit (429)
- Сетевой таймаут при отправке файла в Whisper (файл большой, сеть медленная)
