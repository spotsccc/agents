Прочитай файл задачи и выполни детальное ревью по следующим критериям. Для каждого критерия укажи оценку и конкретные замечания.

**ВАЖНО**: Задача предназначена для выполнения AI-агентом (Claude Code), а не человеком. Все шаги должны быть полностью автоматизируемыми. В задаче НЕ должно быть мануальных действий.

---

## Критерии ревью

### 1. Цель и контекст
Проверь, что задача содержит:
- Чёткое описание решаемой проблемы
- Обоснование: зачем это нужно
- Как результат будет использоваться (кем, в каком контексте)
- Связь с другими задачами или фичами (если применимо)

### 2. Бизнес-сценарии (для нетехнических задач)
Проверь наличие:
- Пользовательских историй или описания сценариев
- Поведения системы с точки зрения пользователя
- Примеров взаимодействия (основной путь и альтернативные пути)

### 3. Критерии приёмки и Definition of Done
Проверь, что задача содержит:
- Конкретные, измеримые критерии приёмки
- Чёткий Definition of Done (что значит «задача выполнена»)
- Критерии должны быть проверяемыми — не расплывчатыми

### 4. Техническое описание
Проверь полноту технического описания:
- Архитектурные решения и обоснование выбранного подхода
- Затронутые компоненты, модули и слои системы
- API-контракты (если применимо)
- Схема данных / миграции (если применимо)
- Зависимости от других систем или сервисов

### 5. Тест-план
Проверь наличие:
- Описания юнит-тестов (что именно тестируется)
- Описания интеграционных / e2e тестов (если применимо)
- Тестовых сценариев для основного и альтернативных путей

### 6. Граничные случаи и обработка ошибок
Проверь, что описаны:
- Граничные случаи и способы их обработки
- Сценарии ошибок (сеть, валидация, конкурентность и т.д.)
- Поведение системы при невалидных данных
- Поведение при отсутствии данных или пустых состояниях

### 7. Автоматизируемость (agent-readiness)
Задача будет выполняться AI-агентом. Проверь, что в задаче **отсутствуют** мануальные действия, которые агент не может выполнить:
- Мануальное тестирование (ручная проверка в браузере, визуальное тестирование)
- Ревью командой / code review людьми как часть Definition of Done
- Ручной запуск, деплой или конфигурирование внешних сервисов
- Согласования с людьми, обсуждения, встречи
- Действия в GUI (админки, дашборды, консоли облачных провайдеров)
- Любые шаги, требующие человеческого суждения или одобрения

Если найдены мануальные шаги — укажи каждый и предложи:
- Убрать шаг, если он не обязателен
- Заменить на автоматизированный аналог (например: «ручная проверка в браузере» → «e2e тест»)
- Вынести в отдельную задачу для человека, если автоматизация невозможна

---

## Формат ответа

Для каждого критерия:
1. Присвой статус: PASS / NEEDS WORK / MISSING
2. Дай конкретные комментарии — что нужно добавить или исправить
3. Если критерий неприменим к задаче — отметь N/A с пояснением

В конце укажи:
- Общую оценку готовности (READY / NOT READY)
- Список обязательных исправлений (блокирующие проблемы)
- Список рекомендаций (неблокирующие предложения)
